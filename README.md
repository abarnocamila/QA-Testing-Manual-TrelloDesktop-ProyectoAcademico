# Testing Manual Funcional - Trello Desktop
**Proyecto académico** - Curso Tester QA – Testing Para Todos - Camila Abarno

## Contexto y Objetivo
Proyecto de testing manual funcional realizado sobre la aplicación Trello Desktop, con el objetivo de validar la funcionalidad crítica Creación de nuevo tablero simulando un entorno de QA basado en requerimientos e historias de usuario.

## Alcance
- Funcionalidad testeada:
- Creación de nuevo tablero  
- Creación de tarjetas dentro de un tablero 

coberturas de pruebas: 
- Validación de reglas de negocio
- Escenarios positivos y negativos
- Validación de inputs
- Persistecia de datos
- Comportamiento de la interfaz
  
## Herramientas utilizadas
- Trello Desktop (aplicativo probado)
- Jira y Zephyr (gestión y ejecución)  
- Notion (documentación inicial)  
- MindMeister (para mapa mental de análisis)  
- Confluence 
- Git y GitHub

## Métricas del proyecto
- Total de casos de prueba: 22
- Casos ejecutados: 22
- Casos exitosos: 20
- Bugs detectados: 2

## Resultados y aprendizajes
Se ejecutaron todos los casos definidos y se detectaron dos incidencias relacionadas con el formato permitido en diferentes campos dentro de la gestión de tableros y tarjetas del aplicativo.  

Este proyecto académico me ayudó a reforzar el diseño de casos con escenarios positivo/negativo, documentación clara y ciclo completo de testing.
